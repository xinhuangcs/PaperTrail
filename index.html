PaperTrail

Authors: Xin Huang · Ruikang Zhao
DTU · Technical University of Denmark
Open-source on Github

PaperTrail is an intelligent literature exploration tool for computer science researchers, especially beginners and interdisciplinary learners.

Based on the arXiv dataset containing over 700,000 computer science papers, PaperTrail helps you go beyond the limitations of simple keyword searches by providing semantically relevant papers tailored to your preferences. Leveraging advanced large language models, it also offers clear reading guidance for each paper, making your paper reading more efficient, precise, and directed.

How PaperTrail works under the hood

Technical overview

PaperTrail is a research prototype integrating unsupervised clustering, hybrid retrieval, recommendation algorithms, and LLM technologies.

Based on an arXiv dataset containing over 700,000 computer science papers, PaperTrail uses OpenAlex to add citation data, enhancing the evaluation dimensions of the literature. Methodologically, PaperTrail implements a multi-stage data processing and recommendation pipeline. During the semantic mining phase, after comparing two approaches – ‘TF-IDF + LSA + K-Means’ versus ‘SBERT + UMAP + HDBSCAN’ – our web page finally adopted the classic TF-IDF + LSA + K-Means combination. For retrieval, we explored three different approaches: TF-IDF, LSA+LSH, and HDBSCAN to rapidly recall the top 100 candidate papers. Our web page finally adopted LSA+LSH. For ranking unlabelled data, we employed a weakly supervised MMR re-ranking algorithm. Pseudo-labels were generated via heuristic rules to learn feature weights, whilst the Maximum Margin Relevance (MMR) algorithm maximised content diversity whilst ensuring relevance. Results were intelligently partitioned into four major categories: Trending, Theory, Application, and Review. Finally, the system integrates Retrieval-Augmented Generation (RAG) technology, employing an LLM strictly constrained by JSON Schema to generate structured learning plan. A closed-loop automated pipeline, built upon GitHub Actions, spans from data updates to front-end feedback, ensuring efficient deployment and instant service delivery.

01
Data Processing: From Raw Text to Latent Semantics

To build a high-quality dataset, we filtered over 700,000 papers in computer science and related fields (such as mathematics and electronics) based on arXiv metadata provided by Kaggle. To address the limitation of missing citation information in the raw arXiv data, we performed large-scale data enrichment using the OpenAlex API. By matching DOIs and titles, we supplemented critical citation data, providing foundational support for the “trust score” in subsequent algorithms. For the constructed dataset, we implemented standardized cleaning procedures: including text concatenation, regularization noise reduction, NLTK stopword filtering, and Porter stemming.

02
Clustering

To overcome the limitations of arXiv's coarse-grained classifications (such as cs.AI and cs.LG), we employ an unsupervised clustering approach to automatically extract more fine-grained thematic structures from the text, enabling precise recommendations that exceed traditional labels. We first employ a manually implemented and rigorously validated TF-IDF algorithm for feature extraction. Subsequently, we utilize LSA to reduce the dimensionality of high-dimensional sparse matrices to a 1000-dimensional latent space through singular value decomposition. LSA effectively captures co-occurrence relationships among words, leveraging the “blurred boundaries” between subfields (e.g., deep learning and reinforcement learning) to uncover cross-domain knowledge with distinct terminology yet intrinsic connections, thereby supporting heuristic recommendations. Finally, by integrating the efficient MiniBatch K-Means algorithm, we rapidly iteratively clustered a large-scale document collection into 40 clusters while maintaining high performance, extracting interpretable topic dictionaries. Additionally, we conducted comparative experiments based on the SBERT+HDBSCAN approach utilizing deep semantic embeddings.

03
Hybrid Search & Recommendation Algorithm

Hybrid Retrieval

For each user query, PaperTrail employs three different retrieval methods to rapidly identify the top 100 most relevant papers for subsequent recommendation steps:

TF-IDF Retrieval: Keyword-based sparse matching using cosine similarity to accurately capture papers containing specific terms.

LSA Retrieval: Maps queries to a low-dimensional semantic space for similarity calculation, effectively retrieving documents with different wording but consistent topics. Incorporates LSH acceleration: Compresses high-dimensional vectors into binary hash codes using random hyperplanes, performing approximate nearest neighbor search within hash buckets to significantly boost speed in large-scale retrieval scenarios.

HDBSCAN Retrieval: Integrates precomputed density clustering results to map documents to semantic clusters (Cluster IDs), while utilizing Apriori-mined associative topic terms to enhance semantic retrieval capabilities.

Recommendation Algorithm

Stage 1: Weak Supervision via Heuristic Labeling

To address the scarcity of manual relevance labels, we employ a heuristic weak supervision strategy. We define view-specific labeling functions (e.g., boosting “survey” terms for Review views) to generate pseudo-labels. These labels construct a set of preference pairs P, serving as a robust proxy for ground-truth relevance.

Stage 2: Pairwise Weight Optimization

We formulate feature weight learning as a convex optimization problem rather than a black-box neural network. We minimize the Regularized Pairwise Logistic Loss to align the weight vector w with our weak supervision signals.

We solve this using Non-negative Coordinate Ascent, ensuring interpretability and rapid convergence—iteratively updating weights to maximize the likelihood of ranking “better” documents higher.

Stage 3: Trust-Aware Scoring and MMR

The final relevance score integrates the learned weights with a source-based Trust Factor τ to penalize low-quality metadata.

To ensure diversity, we apply Maximum Marginal Relevance (MMR) re-ranking, which dynamically balances the learned relevance score against information novelty within the selected set.

04
AI Model

In the final stage, the pipeline transforms raw papers into a Structured Learning Plan. We utilize a versioned System Prompt to strictly enforce a JSON Schema, ensuring the output contains executable modules like Reading Order and relevant advice rather than unstructured text. To guarantee reliability, the system performs double validation: it checks the JSON structure and programmatically verifies that no non-existent papers are hallucinated. Every execution is tagged with a unique Trace ID for full reproducibility. And all artifacts are automatically archived by enabling integration with CI/CD workflows like GitHub Actions.
